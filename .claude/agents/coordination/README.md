# Coordination Agents

Specialized agents for cross-project coordination between mobile and backend teams.

## Available Agents

### ww-aadf-coordinator

**Purpose**: Large-scale coordinated projects with milestone-based execution

**When to Use**:
- Projects with 3+ coordinated tasks across mobile and backend teams
- Milestone-based workflows (Local → Cloud-Dev → Preview → Stakeholder)
- Deployment orchestration between teams
- Task dependency management and parallel execution
- Session recovery for projects exceeding 200k token context

**Example Projects**:
- MVP2 Tranche 1 Foundation & Replanning
- BLE DFU + LoRaWAN Integration
- Authentication System Redesign
- API Migration Projects
- Multi-tenant Feature Rollouts

**System**: Dynamic Cross-Project Coordination System

**Location**: `~/dev/wildlifeai/cross-project-coordination/projects/`

**Key Commands**:

```bash
# Initialize new coordinated project
cd ~/dev/wildlifeai/cross-project-coordination/.scripts
./init-project.sh \
  --slug "project-name" \
  --title "Project Title" \
  --teams "mobile,backend"

# Check project inbox
./check-inbox.sh \
  --project "project-name" \
  --team "mobile"

# Send coordination message
./send-message.sh \
  --project "project-name" \
  --from "mobile" \
  --to "backend" \
  --type "deployment-ready" \
  --message "Preview build complete."

# Check notifications
./check-notifications.sh mobile

# Start project watcher
./watch-project.sh start project-name
```

## Coordination System Architecture

### Two Coordination Modes

**1. Flat-Inbox System** (Day-to-Day)
- **Agent**: `cross-project-coordinator` (existing, at root level)
- **Use**: Simple schema changes, status updates, task requests
- **Location**: `~/dev/wildlifeai/cross-project-coordination/inbox/`
- **Structure**: `inbox/[sender]-to-[receiver]/`
- **Example**: Backend sends schema change notification

**2. Dynamic System** (Large Projects)
- **Agent**: `ww-aadf-coordinator` (this directory)
- **Use**: 3+ tasks, milestones, deployment coordination
- **Location**: `~/dev/wildlifeai/cross-project-coordination/projects/`
- **Structure**: Per-project folders with master plans, inboxes, milestones
- **Example**: MVP2 Tranche 1 with 14+ tasks and cloud-dev deployment

### Decision Matrix

| Scenario | Use Agent | System |
|----------|-----------|--------|
| Backend updates users table schema | cross-project-coordinator | Flat-Inbox |
| Mobile requests new API endpoint | cross-project-coordinator | Flat-Inbox |
| Status update on single task | cross-project-coordinator | Flat-Inbox |
| MVP2 Tranche 1 (14 tasks, milestones) | ww-aadf-coordinator | Dynamic |
| BLE DFU integration (6 tasks, dependencies) | ww-aadf-coordinator | Dynamic |
| Auth redesign (8 tasks, cloud-dev deployment) | ww-aadf-coordinator | Dynamic |

## Workflow Comparison

### Flat-Inbox Workflow (cross-project-coordinator)

```bash
# 1. Check inbox
ls ~/dev/wildlifeai/cross-project-coordination/inbox/backend-to-mobile/

# 2. Read message
cat ~/dev/wildlifeai/cross-project-coordination/inbox/backend-to-mobile/schema-change.md

# 3. Action (e.g., regenerate types)
npm run types:local

# 4. Archive
mv inbox/backend-to-mobile/schema-change.md archive/2025-11/

# 5. Log
.coordination/log-message.sh "Mobile" "Actioned schema change"
```

### Dynamic System Workflow (ww-aadf-coordinator)

```bash
# 1. Check notifications (auto-generated by watcher)
~/dev/wildlifeai/cross-project-coordination/.scripts/check-notifications.sh mobile

# 2. Check project inbox
~/dev/wildlifeai/cross-project-coordination/.scripts/check-inbox.sh \
  --project "mvp2-tranche1" --team "mobile"

# 3. Action (e.g., regenerate types from cloud-dev)
npm run types:cloud-dev

# 4. Archive (automatic via system scripts)

# 5. Send response
~/dev/wildlifeai/cross-project-coordination/.scripts/send-message.sh \
  --project "mvp2-tranche1" \
  --from "mobile" --to "backend" \
  --type "deployment-ready" \
  --message "Preview build complete. Build ID: abc123."
```

## Integration with AADF Ecosystem

The ww-aadf-coordinator integrates with other AADF specialized agents:

- **ww-aadf-mobile-type-guardian**: Type system validation before/after coordination messages
- **ww-aadf-mobile-quality-guardian**: Quality gate enforcement during milestone completion
- **ww-aadf-mobile-dev-agent**: Implementation of coordinated tasks
- **ww-aadf-mobile-test-agent**: Testing during Development and Preview phases
