FOUNDATION LAYER - Task 11: Offline SQLite Foundation

This task creates the complete offline-first data management system that enables the Wildlife Watcher app to function fully offline during field deployments and sync when connectivity is restored.

## Implementation Requirements:

### 11.1: SQLite Database Setup
- Initialize expo-sqlite database (`wildlife_watcher.db`)
- Create comprehensive schema for offline operations:
  - `offline_queue` table for sync operations
  - `local_deployments` table for offline deployment storage
  - `local_projects` table for offline project storage
  - `local_devices` table for device information cache
- Implement database migration system for schema updates
- Set up proper database connection management and cleanup

### 11.2: Offline Service Architecture
- Create `src/services/offline/OfflineService.ts` as main coordinator
- Implement operation queue system for CREATE/UPDATE/DELETE operations
- Add network state monitoring using @react-native-community/netinfo
- Create conflict resolution strategies for data synchronization
- Implement retry logic for failed sync operations
- Add operation deduplication to prevent duplicate syncs

### 11.3: Sync Infrastructure
- Build background sync service that triggers when network becomes available
- Implement incremental sync to minimize data transfer
- Create data integrity validation during sync operations
- Add sync status indicators for user feedback
- Implement sync priority system (critical operations first)
- Handle partial sync failures gracefully

### 11.4: Integration Patterns
- Create offline-first service layer that automatically queues operations
- Integrate with Redux slices for seamless state management
- Add offline indicators throughout the UI
- Implement optimistic UI updates with rollback capability
