FOUNDATION LAYER - Task 11: Offline SQLite Foundation

This task creates the complete offline-first data management system that enables the Wildlife Watcher app to function fully offline during field deployments and sync when connectivity is restored, implementing organisation multi-tenancy and role-based data access patterns per Implementation Spec v1.4.6.

## Implementation Requirements:

### **ENHANCED** 11.1: SQLite Database Setup with Multi-Tenancy
- Initialize expo-sqlite database (`wildlife_watcher.db`)
- **NEW**: Organisation-scoped database schemas
- **NEW**: User role-based data access patterns
- Create schema for offline operations:
  - `offline_queue` table for sync operations
  - `local_deployments` table for offline deployment storage
  - `local_projects` table for offline project storage (organisation-scoped)
  - `local_devices` table for device information cache
  - `local_organisations` table for organisation data
  - `local_user_roles` table for role-based access control
- Migration system for schema updates
- Database connection management

### **ENHANCED** 11.2: Offline Service Architecture with Role-Based Sync
- Create `src/services/offline/OfflineService.ts` with organisation-aware operations
- **NEW**: Maestro test-driven implementation for offline service layer
- Queue system for offline operations with organisation scoping
- **NEW**: Role-based sync filtering:
  - `ww_admin`: Global sync across all organisations and data
  - `project_admin`: Organisation-scoped sync for managed projects
  - `project_member`: Project-scoped sync for accessible projects only
- **NEW**: Organisation context for all offline operations and data isolation
- **NEW**: LoRaWAN status field sync (battery_level, sd_card_usage) with offline caching
- Network state monitoring with organisation priority handling
- Conflict resolution strategies with user role validation
- Retry logic for failed operations with exponential backoff
- Operation deduplication with organisation-based filtering
- **NEW**: Claude Flow SPARC integration for offline architecture patterns

### 11.3: Sync Infrastructure
- Background sync when network available
- Incremental sync to minimize data transfer
- Data integrity validation
- Sync status indicators
- Sync priority system (critical operations first)
- Partial sync failure handling

### **NEW** 11.4: WW Admin Offline Features (MVP User Provisioning)
- **WW Admin User Provisioning**: Offline access to user management across organisations
- Model management offline sync (species, brands, models) for system configuration
- Organisation user list caching with role-based filtering
- **NEW**: User role assignment offline capability (ww_admin, project_admin, project_member)
- System configuration offline access for WW Admin users
- Global admin data synchronization with organisation isolation
- **NEW**: LoRaWAN device configuration offline sync for admin users
- **NEW**: Cross-organisation analytics and reporting offline access
- **NEW**: Bulk user operations offline queue (invite, role changes, deactivation)
