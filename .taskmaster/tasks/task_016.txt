PARALLEL DEVELOPMENT STREAM B - Task 16: End Deployment Flow

This task implements the deployment termination workflow that enables users to properly end active wildlife monitoring deployments and retrieve final data.

## Implementation Requirements:

### 16.1: End Deployment Screen
- Create `src/navigation/screens/deployment/end/EndDeploymentScreen.tsx`
- **Active Deployment Selection**: List user's active deployments with filtering
- **Deployment Information Display**: Show deployment details, duration, location
- **Device Status Check**: Display current device connectivity and battery status
- **Termination Confirmation**: Clear confirmation dialog with deployment impact info
- **Batch Operations**: Option to end multiple deployments simultaneously

### 16.2: Device Reconnection & Communication
- **Automatic Device Discovery**: Attempt to reconnect to deployment device
- **Manual Device Selection**: If automatic fails, allow manual device selection
- **BLE Communication**: Send end deployment command to device
- **Device Status Retrieval**: Get final status data (battery, storage, logs)
- **Connection Fallback**: Handle scenarios where device is unreachable
- **Timeout Handling**: Graceful handling of communication timeouts

### 16.3: End Deployment Service Integration
- Create deployment termination service extending BaseService
- **Status Update to Supabase**: Mark deployment as ended with timestamp
- **Final Data Collection**: Retrieve and store final device metrics
- **Cleanup Operations**: Remove active deployment from local cache
- **Data Integrity**: Ensure deployment data is preserved correctly
- **Offline Support**: Queue termination if device is offline

### 16.4: Completion & Feedback
- **Success Confirmation**: Clear feedback that deployment was ended successfully
- **Data Summary**: Show final deployment statistics and duration
- **Next Steps**: Options to view deployment details or start new deployment
- **Error Recovery**: Handle partial failures and provide recovery options
- **Navigation**: Return to appropriate screen (Maps or Deployments list)

## Workflow Steps:
1. **Select Deployment** → Choose active deployment to end
2. **Device Connection** → Reconnect to deployment device
3. **Final Data Retrieval** → Get final status and metrics
4. **Termination Confirmation** → Confirm deployment end
5. **Status Update** → Update deployment status in backend
6. **Completion Feedback** → Provide success confirmation and options
