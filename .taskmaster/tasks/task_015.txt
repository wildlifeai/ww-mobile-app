PARALLEL DEVELOPMENT STREAM B - Task 15: Start Deployment Flow (6-Step Wizard)

This task implements the complete deployment creation workflow that enables users to start new wildlife monitoring deployments through a guided 6-step process.

## Implementation Requirements:

### 15.1: Project Selection Screen
- Create `src/navigation/screens/deployment/start/ProjectSelectionScreen.tsx`
- **Project Dropdown**: Display user's accessible projects with search/filter
- **New Project Option**: Quick project creation within deployment flow
- **Deployment Naming**: Text input for deployment identification
- **Form Validation**: Ensure project selection and deployment name
- **Continue Navigation**: Progress to device discovery step
- Progress indicator showing Step 1 of 6

### 15.2: New Project Creation (Integrated)
- **Inline Project Creation**: Essential fields only for quick setup
- **Minimal Form**: Name, description, basic settings
- **Integration**: Use existing project creation logic from Task 12
- **Flow Continuation**: Seamlessly return to deployment workflow
- **Validation**: Ensure new project creation is successful

### 15.3: Device Discovery Screen
- Create `src/navigation/screens/deployment/start/DeviceDiscoveryScreen.tsx`
- **BLE Device Scanning**: Integration with existing BLE functionality
- **Permission Handling**: Request Bluetooth permissions gracefully
- **Device List**: Display discovered devices with signal strength
- **Connection Status**: Show device connection and retry logic
- **Device Selection**: Allow user to select target device
- **Manual Refresh**: Button to restart scanning process
- Progress indicator showing Step 3 of 6

### 15.4: Deployment Configuration Screen
- Create `src/navigation/screens/deployment/start/DeploymentConfigScreen.tsx`
- **Location Selection**: GPS location or manual coordinate entry
- **Map Integration**: Visual map showing selected deployment location
- **Use My Location**: Quick GPS-based location selection
- **Capture Method Selection**: Motion detection or timelapse modes
- **Timelapse Configuration**: Interval settings when timelapse is selected
- **Configuration Validation**: Ensure all required settings are provided
- Progress indicator showing Step 4 of 6

### 15.5: Camera Preview Screen
- Create `src/navigation/screens/deployment/start/CameraPreviewScreen.tsx`
- **Live Camera Preview**: Display camera feed from connected device
- **Snapshot Functionality**: Take test snapshots to verify camera angle
- **Image Quality Validation**: Ensure camera is properly positioned
- **Retake Options**: Allow multiple snapshots until user is satisfied
- **User Approval Interface**: Confirm camera setup is acceptable
- Progress indicator showing Step 5 of 6

### 15.6: Final Setup & Deployment Creation
- Create `src/navigation/screens/deployment/start/FinalSetupScreen.tsx`
- **Location Photo Capture**: Photo of deployment site using device camera
- **Description and Notes**: Text area for deployment details and observations
- **Deployment Summary**: Review all configured settings
- **Final Deployment Creation**: Submit deployment to Supabase with offline support
- **Success/Error Handling**: Provide clear feedback on deployment status
- **Navigation**: Return to appropriate screen after completion

## Key Features:

- **BLE Integration**: Seamless device discovery and connection
- **Location Services**: GPS and manual location selection
- **Camera Validation**: Ensure proper device setup before deployment
- **Offline Support**: Deployments can be created without network connectivity

## Wizard Flow:
1. **Project Selection** → Choose project and name deployment
2. **New Project** → (Optional) Create project if needed
3. **Device Discovery** → Scan and select BLE device
4. **Configuration** → Set location and capture method
5. **Camera Preview** → Validate camera positioning
6. **Final Setup** → Add details and create deployment
