PARALLEL DEVELOPMENT STREAM C - Task 19: Maps Integration & Location Services

This task builds upon the basic Maps functionality from the migration to create a comprehensive mapping system that serves as the central hub for Wildlife Watcher deployment management.

## Implementation Requirements:

### 19.1: Maps Screen (Home Screen)
- Create `src/navigation/screens/MapsScreen.tsx` as the main app landing screen
- **Google Maps Integration**: Full integration with react-native-maps
- **User Location Display**: Show user's current location with proper permissions
- **Deployment Marker Visualization**: Display all user's deployments as custom markers:
  - Active deployments (green markers with device info)
  - Ended deployments (gray markers with historical data)
  - Deployment markers show status, battery, and basic info
- **Marker Clustering**: Efficient marker clustering when zoomed out
- **Floating Action Buttons**: 
  - Start Deployment (green) - bottom right
  - End Deployment (yellow) - contextual when near active deployment
- **Map Controls**: Zoom, pan, rotate, map type selection

### 19.2: Location Services Implementation
- **Permission Handling**: Request and manage location permissions gracefully
- **GPS Integration**: High-accuracy location services using expo-location
- **Location Updates**: Periodic location updates for tracking user movement
- **Location Picker Component**: Reusable component for selecting deployment locations
- **Coordinate System Support**: Support for various coordinate formats and conversions
- **Location Validation**: Ensure coordinates are valid and within reasonable bounds

### 19.3: Advanced Map Features
- **Deployment Visualization**: Rich marker information and interaction:
  - Custom markers based on deployment status and device type
  - Marker info windows with deployment details
  - Marker animation for status changes
  - Deployment radius/coverage area visualization
- **Map Data Integration**: 
  - Sync deployment locations with Supabase
  - Real-time updates for deployment marker changes
  - Historical deployment overlay options
- **Navigation Integration**: 
  - Navigate to deployment locations using device GPS
  - Integration with external navigation apps
  - Distance and bearing calculations

### 19.4: Offline Map Support
- **Map Tile Caching**: Cache map tiles for offline access using expo-file-system
- **Offline Area Management**: Allow users to download specific areas for offline use
- **Offline Location Services**: GPS functionality without network connectivity
- **Sync Management**: Queue location updates when offline for later sync
- **Offline Indicators**: Clear indication when operating in offline mode
