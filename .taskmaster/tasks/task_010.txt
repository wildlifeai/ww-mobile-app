FOUNDATION LAYER - Task 10: Core Redux Integration with Supabase

This task extends the existing Redux Toolkit architecture to provide comprehensive state management for all MVP2 features with full Supabase backend integration.

## Implementation Requirements:

### 10.1: Enhanced Auth Redux Slice
- Extend existing `src/store/slices/authSlice.ts` 
- Add user profile data handling beyond basic auth
- Implement session management with refresh tokens
- Add auth state persistence with AsyncStorage
- Handle email confirmation states
- Integrate with Supabase auth service from Task 8

### 10.2: Projects Redux Slice
- Create `src/store/slices/projectsSlice.ts`
- Actions for full CRUD operations (create, read, update, delete)
- Selectors for project lists, details, and filtered views
- Loading and error state management
- Optimistic updates for better UX
- Integration with offline queue system

### 10.3: Deployments Redux Slice  
- Create `src/store/slices/deploymentsSlice.ts`
- Separate active/ended deployment states
- Deployment creation and management actions
- Status tracking and real-time updates
- Location-based deployment filtering
- Device association management

### 10.4: API Integration Layer
- Set up RTK Query integration with Supabase client
- Create `src/store/api/` directory with service endpoints
- Type-safe API calls using generated Supabase types
- Implement caching strategies for offline support
- Error handling and automatic retry logic
- Real-time subscription management
