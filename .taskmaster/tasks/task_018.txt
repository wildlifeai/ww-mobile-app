PARALLEL DEVELOPMENT STREAM C - Task 18: Device Management & BLE Integration

This task builds upon the existing BLE functionality from the migration to provide comprehensive device management capabilities for Wildlife Watcher hardware.

## Implementation Requirements:

### 18.1: Devices Screen
- Create `src/navigation/screens/DevicesScreen.tsx`
- **Device Discovery and Scanning**: Enhanced BLE scanning with user-friendly interface
- **Device List Display**: Show all discovered and paired devices with:
  - Device name, Bluetooth ID, signal strength
  - Connection status (Connected, Available, Offline)
  - Last seen timestamp and device type
  - Battery level and firmware version (when available)
  - Deployment association status
- **Connection Management**: Connect/disconnect devices with status feedback
- **Device Actions**: Test camera, check status, update firmware
- **Persistent Device Storage**: Remember paired devices between sessions

### 18.2: BLE Service Enhancement
- Extend existing BLE functionality from migration phase
- **Connection Reliability**: Implement robust connection management with:
  - Automatic reconnection for lost connections
  - Connection quality monitoring and reporting
  - Connection timeout handling and retry logic
  - Multiple device connection support
- **Device Command Handling**: Enhanced communication protocol support
- **Status Monitoring**: Real-time device status updates and reporting
- **Error Recovery**: Comprehensive error handling and recovery mechanisms

### 18.3: Nordic DFU Integration  
- **Firmware Update Workflow**: Complete integration with react-native-nordic-dfu
- **Update Progress Tracking**: Real-time progress reporting during firmware updates
- **Version Management**: Track device firmware versions and update availability
- **Error Handling**: Robust error handling during firmware update process
- **Update Notifications**: Alert users when firmware updates are available
- **Batch Updates**: Support updating multiple devices efficiently

### 18.4: Device Information & Persistence
- **Device Database**: Store device information in local SQLite database
- **Device Profiles**: Maintain persistent device profiles with:
  - Connection history and reliability statistics
  - Battery usage patterns and trends
  - Deployment history and associations
  - Custom device names and notes
  - Calibration data and settings
- **Sync with Supabase**: Sync device information across user's devices
- **Device Analytics**: Track device performance and usage statistics
