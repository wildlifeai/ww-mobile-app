FOUNDATION LAYER - Task 9: Authentication Screens & Navigation

This task creates the complete authentication user interface and navigation structure for the Wildlife Watcher MVP2 implementation.

## Implementation Requirements:

### 9.1: Login Screen Implementation
- Create `src/navigation/screens/auth/LoginScreen.tsx`
- Email/password form using react-hook-form
- Integration with existing Supabase auth service (`src/services/auth.ts`)
- Form validation (email format, password requirements)
- Loading states, error handling, user-friendly messages
- "Remember me" functionality with AsyncStorage
- Navigate to Maps screen on successful login
- Links to Forgot Password and Sign Up screens

### 9.2: Sign Up Screen Implementation  
- Create `src/navigation/screens/auth/SignUpScreen.tsx`
- Basic fields: email, password, confirmPassword, organization
- Email confirmation workflow integration
- Password strength validation
- Form validation and comprehensive error handling
- Link back to login screen
- Handle pending confirmation state

### 9.3: Forgot Password Screen
- Create `src/navigation/screens/auth/ForgotPasswordScreen.tsx`
- Email input with Supabase password reset integration
- Success/error messaging with clear user feedback
- Return to login navigation
- Handle email confirmation workflow

### 9.4: Navigation Structure Setup
- Update main navigation to handle authenticated/unauthenticated states
- Implement proper routing logic for auth screens
- Session persistence across app restarts using AsyncStorage
- Loading splash screen during auth state check
- Smooth navigation transitions between auth and main app
