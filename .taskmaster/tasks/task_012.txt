PARALLEL DEVELOPMENT STREAM A - Task 12: Projects CRUD Operations

This task implements the complete project management system for Wildlife Watcher, enabling users to create, view, and manage their wildlife monitoring projects.

## Implementation Requirements:

### 12.1: Projects List Screen
- Create `src/navigation/screens/ProjectsScreen.tsx`
- Display project cards with summary information:
  - Project name, description, member count
  - Active deployments count, total deployments
  - Private/public indicator, creation date
- Implement pull-to-refresh functionality
- Add search and filter capabilities (by name, status, date)
- Smooth navigation to project details
- Empty state when no projects exist

### 12.2: New Project Creation
- Create `src/navigation/screens/NewProjectScreen.tsx`
- Multi-section form with react-hook-form:
  - **Basic Information**: name, description, sampling design
  - **Project Settings**: private/public, using bait, monitoring marked
  - **Team Members**: add by email with role assignment
- Comprehensive form validation and error handling
- Integration with Supabase projects table
- Offline support with sync queue integration
- Success/error feedback with navigation

### 12.3: Project Service Layer
- Create `src/services/ProjectService.ts` extending BaseService
- Implement full CRUD operations:
  - `createProject()` - with offline queue support
  - `getUserProjects()` - with computed fields (member count, deployment counts)
  - `updateProject()` - with optimistic updates
  - `deleteProject()` - with confirmation and cleanup
- Member management operations:
  - `addProjectMember()`, `removeProjectMember()`
  - `getProjectMembers()` with profile information
- Type-safe operations using generated Supabase types

### 12.4: RTK Query Integration
- Create `src/store/api/projectsApi.ts`
- Implement optimized queries with proper caching:
  - `getProjects` with tag-based invalidation
  - `createProject` with optimistic updates
  - `updateProject` with immediate UI feedback
  - `deleteProject` with proper cleanup
- Handle loading states and error scenarios
- Integration with offline service for queue management
