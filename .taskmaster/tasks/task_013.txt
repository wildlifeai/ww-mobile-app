PARALLEL DEVELOPMENT STREAM A - Task 13: Project Member Management

This task implements comprehensive team collaboration capabilities for Wildlife Watcher projects, enabling project admins to manage team members and enforce role-based access control.

## Implementation Requirements:

### 13.1: Member Management UI
- Extend project details screen with member management section
- Display current project members with:
  - Profile information (name, email, organization)
  - Role indicators (Admin, Member)
  - Join date and added by information
  - Actions (remove member, change role) for admins
- Add member functionality with email input
- Confirm removal dialogs with proper warnings
- Loading states for all member operations

### 13.2: Permission System Implementation
- Create comprehensive role-based access control:
  - **Project Admin**: Can edit project, add/remove members, delete deployments, change settings
  - **Project Member**: Can create/end deployments, view project data, update own deployments
- Implement UI permission checks:
  - Hide/disable admin-only features for regular members
  - Show appropriate error messages for unauthorized actions
  - Conditional rendering based on user role
- Secure API calls with proper authorization headers
- Backend validation of user permissions before operations

### 13.3: Member Invitation Flow
- Create email-based member invitation system:
  - Validate email addresses before sending invitations
  - Store pending invitations in Supabase
  - Send invitation emails via Supabase Edge Functions
  - Handle invitation acceptance/rejection
- Pending invitation management:
  - Display pending invitations in member list
  - Allow admins to resend or cancel invitations
  - Automatic cleanup of expired invitations
- Integration with user registration flow for new users

### 13.4: Member Activity & Permissions Integration
- Track member activity within projects:
  - Last active timestamp
  - Recent deployment activity
  - Permission change history
- Implement permission enforcement throughout app:
  - Navigation restrictions based on role
  - Feature availability based on permissions
  - Error handling for permission violations
- Admin tools and project oversight capabilities
