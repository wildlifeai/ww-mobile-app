## Purpose

Reference links serve multiple purposes:

1. **Navigation**: Quick access to key documents from coordination hub
2. **Context**: Each reference provides summary and "when to use" guidance
3. **Related Docs**: Cross-references to related coordination materials
4. **Avoid Duplication**: Single source of truth for implementation guides

## When to Add New References

Create reference links for:
- Implementation guides used by coordination workflows
- Task-specific files referenced in coordination messages
- Backend documentation needed for mobile development
- External documentation critical to cross-project work

## Reference File Structure

Each reference should include:
- **Location**: Absolute path to original document
- **Purpose**: 1-2 sentence summary
- **When to Use**: Specific use cases
- **Related Documents**: Cross-references to other coordination materials
- **Link**: Relative path to full document
